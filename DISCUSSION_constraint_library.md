# 制約ライブラリ構想 (Discussion in Progress)

Date: 2026-04-25

## 出発点
AI向けライブラリはどうあるべきか。

## 到達した概念

### ライブラリはツールではなく知識である
- ライブラリは計算しない。変数間の関係性を宣言する
- どの変数を固定するかは呼び出し側が決める。入力/出力の区別がない
- 制約を入れると、狭まった関係性が返る。値が返るのは自由度ゼロの特殊ケース
- 順方向も逆方向もない。制約を積むだけ

### 再帰的構造
- 知識 + 制約 → 狭まった知識
- 狭まった知識 + 制約 → さらに狭まった知識
- ... → 最終的に値（自由度ゼロ）
- Event Sourcing と同型：イベントを積むと状態が確定する

### バグが減る構造的理由
1. 関数が1つなら選択ミスが起きない
2. 純粋なら呼び出し順序の依存がない
3. 内部状態がないから想定のズレがない
4. 小さい共有関数がないから変更時の影響範囲の見落としがない
5. 矛盾は空集合として自然に検出される

### 人間との役割分担
- 人間：関係性の宣言、意図の伝達
- AI：制約系の構築、固定の適用、結果の説明
- 人間がライブラリを直接触る必要はない。AIが間に立つ

## 試作: 3人じゃんけん
- `constraint-janken.js` で概念実証済み
- 27の可能世界を知識として持ち、制約でフィルタする
- 順方向（手→結果）も逆方向（結果→手）も同じ操作で動作
- 矛盾検出も動作

### 答えではなく知識を返す
- 従来の道具（計算機、API、DB、ML）は全て「質問→答え」
- この構想は「質問→狭まった知識」。答えは自由度ゼロの特殊ケース
- 答えは行き止まり。知識は続きがある
- AIは狭まった知識を見て、次にどの制約を足すべきか判断できる

### スケーリング：モンテカルロ法
- 全列挙が不可能な規模では、ランダムサンプリングで制約に合う世界を集める
- サンプル群からパターンを推測するのはAIの得意分野
- 確信度もパラメータになる（サンプル数で精度と速度のトレードオフ）

### AIの思考の透明性・信頼性
- AIが制約ライブラリを通して推論すれば、各段階の狭まり方が記録として残る
- 「いきなり答えに飛んだ」か「段階的に狭まった」かで推論の信頼性を構造的に評価できる
- テキストの説明は後付けで嘘をつける。制約の狭まり方は構造だからごまかしにくい
- 人間が同じ制約を同じ順番で追体験できる。説得ではなく追体験による納得

### 教育への応用
- 知識が段階的に狭まる過程がそのまま説明になる
- 答えだけ教えるのではなく、構造が見える
- 制約を足す順番を変えれば、同じ知識から異なる切り口で教えられる
- §0.0の逆：AI向けに作ったものが、人間の学習にも使える

### 同じ知識、異なる消費
- 知識は複雑なまま、何も隠さずに保存する（情報の欠落がない）
- AI向け：複雑なまま受け取る。AIにとって複雑性は問題ではない（§0.0）
- 人間向け：制約を足して具体例を生成する。人間は具体例を多く見て暗黙知に変える
- 知識は一つ。消費の仕方が違うだけ

### 制約と隠匿の同一性
- 人間の「理解する」は本質的に「ほとんどを隠して一片だけ見る」こと
- 抽象化、カプセル化、関数分割、具体例 — 全て隠匿の形態
- 人間には隠匿が必要。全部見えたら理解できない
- 制約を足す = 意図的に隠匿する = 人間が覗ける断片を作る
- §0.0の「隠匿は人にとって問題ではない」の真意：人間の知的活動は隠匿の上に成り立っている

### 応用: L1→L2の意図絞り込み
- L2(Intent層)の本質は「変換」ではなく「絞り込み」だった
- 全てのCommand（可能な意図の全体）から、生操作を制約として積み、意図を絞り込む
- if文の分岐が消える。分岐を書かないからバグが出ない
- 条件の書き漏らし → バグではなく「未確定」が返る。壊れない
- 条件の重複・順序ミス → 分岐がないので構造的に起きない
- バグの修正が「コードの書き直し」ではなく「知識の追加」になる
- 入力はバグの温床（想定外の操作、デバイス差異、タイミング、組み合わせ爆発）だが、制約ライブラリなら捕まえ損ねても壊れない

### 応用: 入力の誘導とバリデーション不要化
- 未確定は空白ではなく、残った可能性の集合
- 残った可能性から「次に何を入力すべきか」を提案できる
- 入力順序を強制しない。わかるところから埋めればいい
- 選択肢が自動的に絞られるため、ありえない組み合わせを入力できない
- バリデーションが不要になる

### この仕組みから導出されるもの一覧
1. バグが構造的に減る（分岐を書かない）
2. 矛盾を自動検出（空集合）
3. 曖昧さを可視化（未確定が見える）
4. AIの思考が透明になる（狭まり方の記録）
5. 人間に教えられる（具体例の生成）
6. 次の入力を提案できる（残りの可能性）
7. 入力順序を強制しない（どこからでも）
8. バリデーション不要（ありえない入力が不可能）
9. バグ修正が知識の追加（コード書き直し不要）
- 全て「知識 + 制約 → 狭まった知識」という一つの仕組みから

### 応用: 会計ソフト
- 会計は本質的に関係性の塊（借方=貸方、資産=負債+純資産、利益×税率=税額）
- 順方向だけでなく逆方向に問える（「税額をX以下にするには利益をいくらに？」）
- 今の会計ソフトは全部順方向。入力→結果のみ。逆方向に問えない
- 税率テーブル、業種分岐 → if文ではなく知識として宣言
- 入力フォーム → 順序自由、矛盾は自動検出
- 会計士が頭の中でやっている逆算・シミュレーションが仕組みとして実現する
- kaikei プロジェクトとの親和性が高い

### 応用: 暗黙知の外部化
- 人間の専門家が持つ暗黙知（会計士、医者、職人等）は言語化しにくいが、関係性としてなら記述できる
- 制約ライブラリに専門知識を入れれば、AIがそれをツールとして常時使える
- 学習ではなく参照。知識は検証可能・更新可能・説明可能
- 今のAIの知識はモデルの重みの中にあって取り出せない。制約ライブラリの知識は外にあって見える
- ブラックボックスのモデルではなく、透明な関係性として専門知識を持てる

### 応用: 倫理・法律の形式化
- 倫理判断を関係性として明示的に記述できる。なぜその判断かが追体験可能
- 今のAIの倫理は重みの中にあり検証不能。制約ライブラリなら議論して修正できる
- 法律は本来「関係性の束」。制約ライブラリで書けば曖昧さがなく矛盾も自動検出される
- 新しい法律が既存の法律と矛盾するか即座にわかる

### 応用: AIスキル/ツールの最終形
- 今のAIツール：関数を呼ぶ → 答えが返る → ブラックボックス
- 制約ライブラリがツールになれば：知識を参照 → 制約を積む → 過程が全部見える
- ツールから知識へ。AIの能力拡張の仕方が根本的に変わる

### 判断の主権を人間に戻す
- AIを信じるかどうかの問題が、知識を信じるかどうかの問題に変わる
- 知識は読める、検証できる、専門家が指摘できる、直せる
- 人間はAIの結論に対して、同じ知識に自分の制約を入れて検証できる
- AIと人間が同じ知識を共有し、それぞれが制約を足して対話できる

### 応用: ゲーム
- ゲームは本質的にルールの塊。ルール＝関係性だから制約ライブラリと相性が最高
- 戦闘システム：パラメータ群→ダメージ。逆方向に「この敵を倒すには？」と問える
- クエスト/ストーリー分岐：「このエンディングに行くには何が足りない？」
- クラフト/レシピ：「この武器を作りたい」→ 必要素材が出る
- バランス調整：パラメータ変更の影響を逆方向に探れる
- 文脈依存の複雑な関係性（同じ行動でも状況で結果が変わる）をif文なしで自然に表現できる
- AIの敵が「この状況で何をすべきか」を制約から導ける

## 懸念: 透明性の両面
- この仕組みでAIの行動制限（ガードレール）を定義できる
- 制約として定義すれば網羅的で、穴も可視化できる。今のif文ベースのガードレールより強力
- しかし「ガードレール」は定義次第で「安全のための制限」にも「検閲」にもなる
- 制約ライブラリの知識は透明で検証可能。何を制限されているかも見える
- 透明性は制御する側にも制御される側にも等しく働く
- 誰がこの知識を書くかで意味が変わる（安全の専門家、政府、企業）
- これは仕組みの問題ではなく、誰が使うかの問題
- 悪用する者はどの道具でも悪用する。懸念を記録しておくこと自体が透明性の実践である
- ただし構造的には防御側が有利：攻撃は全ての制約を同時に回避する必要があるが、防御は制約を一個足すだけで穴を塞げる
- プロンプトハック対策としても有効：今のハックは「テキストでテキストを上書き」するから成立する。制約は構造であり、テキストで上書きできない
- 重みの中の不透明な制御（洗脳）より、外部化された透明な制御（教典）の方が、制御する側にも制御される側にもましである

### 応用: 知識経済
- 制約ライブラリ自体が商品になる。専門家の暗黙知が売買可能な資産になる
- 「熟練会計士の30年の判断知識」「業界の法規制の関係性」を制約ライブラリとして売れる
- 買った側はAIに読ませるだけで即座にその専門知識を使える
- 検証可能だから品質が評価できる。買う前にAIが矛盾チェックできる
- 専門家が引退しても知識がファイルとして残る

### 応用範囲の結論
- 個別に列挙する意味がない。AIが使える分野全てで使える
- 「使えない分野」を探した結果：真にランダムなもののみ
- しかし真にランダムなものに対しても「狭まらないこと」自体が情報になる（乱数品質テスト）
- 結論：汎用的な知識表現形式である

## 未解決の論点
- 全列挙は小規模でしか成立しない。規模拡大には制約伝播・数式処理・モンテカルロが必要（実装の問題であり概念の問題ではない）
- 非数値（論理、構造）の制約にも拡張できるか
- 出力の表現形式をどうするか
- バイブルに入れるとしたらどのセクションか

## 関連するバイブル原則
- §0.0 認知特性の非対称性: 人間に使いこなせなくても、AIに使いやすければいい
- §0.1 ローカリティの極大化: 関係性が1箇所に閉じている
- §0.2 AI Drives All: AIが作り、AIが使う
