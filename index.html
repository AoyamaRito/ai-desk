6つの柱 (Core Philosophy)

人間向けの「良い設計（隠蔽・カプセル化）」は、AIの推論に必要な文脈を奪う。この非対称性を前提に、AIのポテンシャルを最大化する原則です。

1. 認知の非対称性

複雑性は人の問題、隠匿はAIの問題。カプセル化を捨て、情報を平坦に展開する。

2. 重厚関数 (Heavy Functions)

共通ヘルパーを禁止し、関数を長く保つ（インライン化）。Emblemタグによる仮想境界で区切る。

3. 条件畳み込み (Constraint Folding)

if/else の分岐ツリーを捨て、全可能世界を生成してから制約フィルターで結果を導出する。

4. 鉱脈採掘型開発

ロジックを人間が書くのではなく、ランダム探索(モンテカルロ)で「正解」を掘り出し、AIが法則を合成する。

5. 複式数学 (3Dplus)

GPU出力などのブラックボックスに対し、並走する純粋関数（Twin）で座標・状態を検算・証明する。

6. 自律観測 (AI-Eyes)

AIが人間に報告を求めず、自律的に画面・エラーを観測し、ステップ実行でデバッグするインフラ。

提供するツール (Tools)

AIがコードを編集する「手」と、実行結果を観測する「目」。

✍️ ai-desk.js (手)

巨大ファイルを Emblem タグで仮想分割し、AIがコンテキストを失わずに局所更新するための CLI ツール。

node ai-desk.js file.js skeleton
node ai-desk.js file.js focus Name
node ai-desk.js file.js apply patch.js
node ai-desk.js file.js check

👁️ ai-eyes.js (目)

エラー自動収集、HTML/Canvasスナップショット保存、動画記録、AIからのリモート操作を受け付けるゼロ依存サーバー。

# AIによる自律デバッグ例
curl -X POST localhost:3000/input \
  -d '{"action":"eval", "code":"..."}'
curl localhost:3000/snapshot/request
